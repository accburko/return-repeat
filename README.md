# Optymalizacja procesu zwrotów - Projekt "Return-Repeat"

Projekt automatyzacji procesu obsługi zwrotów, mający na celu wyeliminowanie pracy manualnej i poprawę przepływu informacji (workflow).

## Cel projektu
Przejście z modelu rozproszonego (Google Chat, zadania ręczne) na zintegrowaną aplikację obsługującą kolejki i statusy w czasie rzeczywistym.

### Porównanie procesów

| Funkcja | Obecnie (Manualnie) | Propozycja (Aplikacja) |
| :--- | :--- | :--- |
| **Przepływ danych** | Ręczne przekazywanie | Centralna baza w aplikacji |
| **Komunikacja** | Google Chat (wątki, chaos) | Automatyczne kolejki zgłoszeń |
| **Taski** | Tworzone ręcznie | Automatyczne przypisywanie |
| **Logistyka** | Szukanie produktów na hali | Przypisane miejsca na wózkach |
| **Transparentność** | Brak podglądu statusu | Status widoczny dla wszystkich live |
| **Analityka** | Brak danych | Raporty i statystyki |
